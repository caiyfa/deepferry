概览：双模式入口与三层架构

North Star · 16.5 周 / 4 里程碑

deepferry 让任意 LLM Agent 通过统一的 MCP 协议查询 MySQL、PostgreSQL、HTTP API （含两步认证），并在桌面表格应用中查看结果。产品 = ferry（在不同数据源间主动搬运并重组数据）。 下方为整体架构，所有 spec 场景均围绕此架构展开。

04

设计原则与一致性约束

跨所有场景

🧭 交互与视觉一致性

  • 状态三色：成功绿 / 重试或慢琥珀 / 失败红，桌面与 Web 面板一致
  • 错误永远结构化：JSON {code, message, suggestion}，绝不向 Agent/用户暴露 Python traceback
  • 所有数据源视觉等价：MySQL / PG / HTTP / 编排源在同一列表，无 isinstance 特判
  • 琥珀色 = auth_retry 专用（D4 / 历史 / Dashboard），与普通成功/失败区分
  • 脱敏：span attributes 中的 token / 密码统一遮蔽

📐 布局模式

  • 桌面应用：左固定侧边栏(188px) + 主画布；macOS 红黄绿圆点标题栏
  • Web 面板：深色侧边栏 + 浅色画布（与桌面区分入口）
  • 表格：统一 .gt 样式，可排序列头带 ↕，过滤行内嵌
  • Span 树：深度缩进 + 瀑布条(timeline) + kind 色块 + 耗时列
  • 统计卡片：mini-stat 复用于 Dashboard / 结果页 / 执行详情

🛡️ 生产护栏（每条数据路径强制）

  • 只读强制（read-only enforcement）
  • 语句超时（statement timeouts，默认 30s）
  • 行数上限（row caps，默认 10000）
  • 连接池上限（pool ceilings）
  • 过滤下推强制（filter-pushdown，DuckDB）
  • 内存有界（memory bounding）
  • 流式游标（streaming cursors）

🎯 验收锚点（节选自 spec）

  • macOS 双击安装 → sidecar 自启；Win 同等验证
  • 10 万行滚动 <100ms；列排序/过滤正确
  • 连接测试：合法绿勾 / 非法红叉 + 消息
  • 50 span execution <500ms 渲染（虚拟列表）
  • 删除数据源 → config 移除，历史保留
  • config.toml 改动 → Agent 下次查询生效

🔀 双模式分层原则

  • 同一查询路径：Desktop 与 Server 共用 DataSource ABC + config.toml + ${ENV_VAR} 注入 + 审计脱敏
  • 差异仅在：谁配置源（Agent propose_source vs admin config-panel）+ 凭据来源（keychain vs Vault/KMS）+ Agent 工具集（有/无 propose_source）
  • Server 模式：config-panel Web UI 是源生命周期管理面；非 admin 可浏览查询，不可增删改
  • 热重载共享：两模式 CRUD 均经 registry.refresh() 原子指针交换，在途查询不受影响

🔐 凭据绝不穿越 LLM 上下文（硬规则）

  • propose_source 无 password 字段——仅提交拓扑（type/host/port/database/name）
  • list_sources / query 返回拓扑，绝不返回凭据
  • DataSource 经 ${ENV_VAR} 解析凭据，绝不接受 Agent 输入的密码
  • 审计 span 脱敏 password|secret|token|api_key → [REDACTED]
